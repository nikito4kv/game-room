# Спецификация проекта «Game Room» (рабочее название)

> Голосовая игровая комната без регистрации в эстетике CS 1.6.
> Дата: 2026-06-23. Версия документа: 1.0 (черновик по итогам discovery-интервью).

---

## 1. Краткое резюме

Веб-приложение, где компания друзей за пару секунд собирается в общей «комнате»
для совместной игры: голосовой чат, демонстрация экрана и доска для рисования
тактик. Вход без регистрации — по коду комнаты (опционально с паролем).
Лёгкая, быстрая и более «игровая» альтернатива голосовому каналу Discord,
с фирменным визуальным стилем CS 1.6, но современным удобным интерфейсом.

---

## 2. Проблема

Сейчас друзья играют вместе (Rocket League и др.) и сидят в голосовом канале
Discord. Discord требует аккаунт, сервер, заранее настроенные каналы — это
лишнее трение для разовой совместной игры, и в нём нет инструмента, чтобы прямо
во время игры рисовать тактику.

**Боль:**
- Высокий порог входа (регистрация, сервер, каналы) для простого «собраться поиграть».
- Нет встроенного инструмента для разбора тактик/стратегий (доска).
- Стандартный «корпоративный» вид Discord — без игровой атмосферы.

**Почему сейчас:** есть готовые медиа-платформы (LiveKit) с бесплатным стартом,
которые позволяют сделать такой продукт силами одного разработчика + нейросети,
без своей сложной WebRTC-инфраструктуры.

---

## 3. Критерии успеха

- **MVP-успех:** автор и его друзья реально пользуются комнатой вместо Discord
  во время игр (голос стабилен, экран и доска работают).
- **Продуктовый успех (рост):** в продукт заходят незнакомые компании,
  возвращаются повторно; растёт число активных комнат в неделю.
- **Технический успех:** запуск и первые месяцы — в пределах бесплатных лимитов
  (без обязательных расходов на инфраструктуру).

Метрики для отслеживания (после запуска): число созданных комнат/день,
среднее число участников в комнате, длительность сессий, доля возвратов.

---

## 4. Пользователи (персоны)

- **Хост (создатель комнаты).** Геймер, технически средний уровень. Создаёт
  комнату, кидает друзьям код/ссылку, модерирует, рисует тактику на доске.
- **Участник.** Друг хоста или человек из публичного списка. Заходит по коду/ссылке,
  вводит ник, общается голосом, смотрит экран/доску.

Все — десктоп, браузер (Chrome/Firefox на Windows в первую очередь).

---

## 5. Пользовательский путь (основной сценарий)

1. Пользователь открывает сайт. Видит ретро-экран в стиле CS 1.6:
   кнопки «Создать комнату» / «Войти по коду» и (опционально) список публичных комнат.
2. **Создание:** вводит ник и название комнаты, при желании ставит пароль и галочку
   «показать в публичном списке». Получает короткий код/ссылку, чтобы позвать друзей.
3. **Вход:** вводит код (и пароль, если есть) + ник → попадает в комнату.
4. **В комнате:** браузер просит доступ к микрофону. Пользователь говорит,
   видит участников, управляет своим микрофоном и общим звуком; может включить
   демонстрацию экрана; может открыть доску тактик и рисовать.
5. **Модерация:** хост может кикнуть/«забанить» участника (в пределах сессии),
   заглушить чужой микрофон, передать права хоста, управлять доской.
6. **Завершение:** когда последний участник вышел — комната автоматически удаляется.

**Ошибки/крайние случаи:** неверный код/пароль; нет доступа к микрофону;
имя/комната заняты; разрыв соединения и переподключение; хост вышел
(права переходят к следующему участнику автоматически или вручную).

## 6. Функциональные требования

### P0 — Must Have (MVP)
- **Создание комнаты** с ником, названием, опциональным паролем и флагом публичности.
- **Вход в комнату по коду** (+ пароль, если задан) с вводом ника.
- **Голосовой чат** (WebRTC через LiveKit) для всех участников комнаты.
- **Управление звуком:** вкл/выкл своего микрофона, вкл/выкл общего звука (заглушить всех).
- **Демонстрация экрана** (screen share) — показать свой экран комнате.
- **Доска тактик — базовая:** свободное рисование (кисть, цвета, ластик) поверх
  фона; возможность загрузить картинку-карту как фон. Рисунки синхронизируются
  всем в реальном времени (через data-канал LiveKit).
- **Модерация хоста:** кик/бан (в пределах сессии), заглушить чужой микрофон,
  передать права хоста.
- **Авто-удаление комнаты**, когда вышел последний участник.
- **Настройки устройств:** выбор микрофона и устройства вывода звука, регулировка
  громкости.
- **Визуальный стиль CS 1.6** с современным UX (см. раздел 9).

### P1 — Should Have (сразу после MVP)
- **Доска тактик — расширенная:** готовые фигуры и стрелки, текст, перемещаемые
  метки игроков по полю.
- **Публичный список комнат:** витрина комнат с флагом «публичная», поиск/фильтр.
- **«Память» браузера:** запоминание ника и настроек устройств (localStorage).
- **Индикация говорящего** (подсветка активного микрофона), индикатор качества связи.

### P2 — Nice to Have (позже)
- Опциональные **аккаунты** (для нормального бана, сохранённого профиля, списка друзей).
- Текстовый чат в комнате.
- Запись сессии / сохранение доски.
- Мобильная адаптация (сейчас вне области).
- Монетизация (премиум-комнаты/функции).

## 7. Техническая архитектура

### Стек
- **Frontend + Backend:** Next.js (React + TypeScript), App Router.
- **Хостинг:** Vercel (бесплатный тариф).
- **Медиа (голос + демонстрация экрана + data-канал доски):** **LiveKit**.
  - Старт: **LiveKit Cloud**, бесплатный тариф (5 000 WebRTC-минут/мес,
    до 100 одновременных подключений, без кредитной карты).
  - Путь роста: **self-hosted LiveKit** (open-source, Apache 2.0) на своём VPS —
    тот же клиентский код, без привязки к ценнику облака.
- **SDK:** `@livekit/components-react` + `livekit-client` (фронт),
  `livekit-server-sdk` (генерация токенов на бэке).

> Выбор стека продиктован тем, что разработка ведётся с помощью нейросети:
> Next.js + React + TS — самый распространённый и хорошо документированный стек,
> у LiveKit лучший SDK именно под React.

### Данные и состояние
- **На старте — без собственной базы данных.** Источник правды об активных
  комнатах и участниках — сам LiveKit (его Server API / room metadata).
- **Метаданные комнаты** (название, флаг публичности, хэш пароля, кто хост)
  хранятся в `metadata` комнаты LiveKit.
- **Ник и настройки устройств** пользователя — в `localStorage` браузера.
- **Состояние доски** передаётся в реальном времени через data-канал LiveKit
  (не персистится после удаления комнаты в MVP).
- Если позже понадобится персистентность (аккаунты, история) — добавляется
  лёгкая БД (например, Postgres на бесплатном тарифе) без переписывания медиа-части.

### Серверная логика (Next.js API / Route Handlers)
- **Генерация токена доступа** в комнату LiveKit (никогда не на клиенте —
  секретный ключ только на сервере).
- **Проверка пароля** комнаты перед выдачей токена.
- **Создание комнаты** и запись её метаданных.
- (P1) **Список публичных комнат** — через LiveKit Server API (`listRooms`) с фильтром по metadata.

### Безопасность
- LiveKit API-ключ/секрет — только в серверных переменных окружения (env), не на клиенте.
- Пароль комнаты хранится хэшированным, проверяется на сервере.
- Токены LiveKit — короткоживущие, привязаны к конкретной комнате и правам.
- **Известное ограничение:** без аккаунтов «бан» работает только в пределах
  сессии/устройства и легко обходится (новый ник). Полноценный бан возможен
  только с введением аккаунтов (P2).

## 8. Нефункциональные требования

- **Производительность:** комфортный голос с задержкой в норме для WebRTC;
  демонстрация экрана — приемлемая плавность для разбора (не обязательно 60fps).
- **Масштаб:** до 2–6 участников в комнате (целевой максимум), много параллельных комнат.
  Архитектура LiveKit позволяет вырасти без смены клиентского кода.
- **Стоимость:** запуск и ранний рост — в пределах бесплатных лимитов LiveKit Cloud + Vercel.
- **Надёжность:** переподключение при обрыве связи; корректная передача прав хоста.
- **Платформа:** десктопные браузеры (Chrome/Firefox), Windows в приоритете.

---

## 9. Дизайн и UX

- **Визуальный язык:** эстетика CS 1.6 — характерные зелёные рамки/панели
  («лифтовый» зелёный), угловатые окна, фирменный пиксельный шрифт в заголовках/акцентах.
- **Принцип:** стиль — ретро, но **поведение и удобство — современные**.
  Никаких неудобных «старых окон»: понятные кнопки, отзывчивость, нормальные
  состояния загрузки/ошибок, аккуратные подсказки доступа к микрофону/экрану.
- **Шрифт:** пиксельный — для заголовков и акцентов; для длинного/мелкого текста —
  читаемый современный шрифт (чтобы не страдала разборчивость).
- **Экраны:** лендинг (создать/войти/список), экран комнаты (участники, панель
  управления звуком, демонстрация экрана, доска), настройки устройств.

---

## 10. Вне области (Out of Scope для MVP)

- Регистрация и аккаунты (только ник).
- Мобильная версия / адаптив под телефон.
- Текстовый чат, запись звонка, сохранение доски между сессиями.
- Монетизация.
- Полноценный бан с защитой от обхода.

## Приложение А. Итоги исследования медиа-платформ (июнь 2026)

Сравнивались LiveKit, Daily.co, Agora, 100ms, Twilio Video.

| Платформа | Бесплатно/мес | Карта | Демо экрана | Self-host бесплатно | Цена за 1000 мин | Data-канал |
|---|---|---|---|---|---|---|
| **LiveKit** | 5 000 мин, до 100 подкл. | Нет | Да | **Да (Apache 2.0)** | **$0.50** | Да, нативно |
| Daily.co | 10 000 мин | Нет | Да | Нет | $4.00 | Да |
| Agora | 10 000 мин (общие) | — | Да | Нет | $0.99–3.99+ | Через RTM |
| 100ms | 10 000 мин | Нет | Да | Нет | $4.00 | Да |
| Twilio Video | мало | Да | Да | Нет | дороже | Да |

**Вывод:** выбран **LiveKit** — единственный, кто совмещает щедрый бесплатный
старт без карты, нативные data-каналы (транспорт для доски), отличный React/JS
SDK и бесплатный open-source self-hosting. Это даёт реальный путь «запуститься
бесплатно → вырасти» без смены кода. Agora опасен моделью оплаты по агрегатному
разрешению при групповом видео; Twilio Video дважды анонсировал закрытие —
рискованно для нового проекта.

Источники: livekit.com/pricing, github.com/livekit/livekit, daily.co/pricing/video-sdk,
agora.io/en/pricing, 100ms.live/pricing.

## Приложение Б. Идеи по раскрутке (go-to-market)

- **Ниша конкретных игр.** Заходить не «во всех геймеров», а в сообщества Rocket
  League и других командных игр (Reddit, тематические Discord-серверы, VK/Telegram-чаты).
  Месседж: «собрал друзей в один клик + рисуй тактику».
- **Доска тактик — главный отличитель.** Это то, чего нет в Discord. Делать вокруг
  неё контент: короткие ролики «разбор тактики на доске», шаблоны карт.
- **Ретро-эстетика как вирусный повод.** Стиль CS 1.6 — ностальгия, ей хочется
  делиться. Красивые скриншоты/гифки интерфейса — органический шеринг.
- **Нулевое трение = легко позвать.** «Кинул ссылку — друг сразу в комнате».
  Это снижает порог приглашения и работает как естественный рост (k-фактор).
- **Идеи названий (на выбор):** обыграть «комнату», «лобби», «фраг», ретро-мотив
  CS. Финал — за автором.

## Приложение В. Известные ограничения / риски

- Без аккаунтов: слабый бан, нет удержания/возврата по умолчанию — заложен путь
  добавления аккаунтов (P2), архитектура к нему готова.
- Бесплатные лимиты LiveKit Cloud — хард-кап; при росте нужен переход на платный
  тариф или self-host (план роста описан).
- Демонстрация экрана + голос на 2–6 человек укладывается в возможности LiveKit;
  для больших комнат (вне текущей области) потребуется отдельная проработка.
