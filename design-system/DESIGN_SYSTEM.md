# Game Room — Дизайн-система

Сменяемая дизайн-система для голосовой игровой комнаты. Бесплатная тема по
умолчанию — **«Arena»** (тёмный киберспорт). Движок спроектирован так, чтобы в
будущем можно было докупать и переключать **полные иммерсивные скины** (цвета,
шрифты, форма, текстуры, звуки, курсоры, анимации).

> Файлы рядом: `tokens.css` — готовый контракт токенов и скин Arena;
> `preview.html` — живой показ (открой в браузере, переключай скины вверху).

---

## 1. Идея и характер

Приложение — это «комната, где собираются играть с друзьями». Дизайн должен:

- **с первого взгляда читаться как игровое** — узнаваемо, с характером, чтобы
  захотелось показать друзьям (визуальный контраст = вовлечение);
- **не утомлять** — впятером в нём сидят часами, поэтому база спокойная и
  тёмная, а «игровое» живёт в акцентах, свечении говорящего, иконках и шрифте;
- **быть «движком», а не одной темой** — внешний вид меняется сменой набора
  токенов, а не переписыванием компонентов.

**Принципы**

1. **Тёмная база, неоновый акцент.** Глубокий холодный фон, один яркий бренд-цвет.
2. **Свет = жизнь.** Свечение и цвет показывают активность: кто говорит, что в
   эфире, кто хост. Тишина — спокойная, событие — светится.
3. **HUD-типографика.** Заголовки и «системные» подписи — техничный шрифт
   (Exo 2); основной текст — нейтральный и читаемый (Inter).
4. **Форма с лёгкой остротой.** Умеренные скругления, тонкие рамки-волоски,
   аккуратные тени. Не «корпоративно-мягко», но и не «играем в кубики».
5. **Сдержанность по умолчанию.** Дефолт скромный; «громкость» (текстуры,
   звуки, фон-арт) включают платные скины.
6. **Игровой HUD-каркас.** Узнаваемость даёт не «громкость», а форма: скос
   углов, угловые скобки-«прицел», keycap-подсказки клавиш, лента событий
   (киллфид). Из этих деталей собирается «игровой» характер без перегруза.
7. **Доступность — это фича, а не галочка.** Цвет всегда продублирован иконкой
   и подписью; текст ≥14px; движение отключаемо; всё проходимо с клавиатуры.

## 2. Токены — три слоя

Вся система держится на CSS-переменных (поверх Tailwind v4 `@theme inline`,
который уже есть в `src/app/globals.css`). Три слоя:

| Слой | Что это | Кто использует |
|------|---------|----------------|
| **1. Примитивы** | Сырые значения палитры (`--c-iris-500: #6e66ff`) | Только слой 2. Компоненты — нет. |
| **2. Семантика** | Смысловые токены (`--accent`, `--surface`, `--live`) | Все компоненты. **Это контракт скина.** |
| **3. Форма/тип** | Шрифты, радиусы, тени, свечения, текстуры, курсор, движение | Все компоненты. Тоже под скином. |

**Почему так:** компонент знает только «дай мне `--accent`», а *чем* является
акцент в конкретном скине — забота скина. Поэтому новый скин не трогает ни
строчки в компонентах — он лишь заполняет контракт.

### Контракт семантики (минимум, который заполняет каждый скин)

```
Поверхности:  --bg  --bg-deep  --surface  --surface-2  --inset
Рамки:        --border  --border-strong
Текст:        --text  --text-dim  --text-mute
Акцент:       --accent  --accent-hi  --accent-press  --on-accent
Состояния:    --live  --live-hi  --host  --danger  --warn  --info
Свет/фокус:   --ring  --glow-accent  --glow-live
Форма/тип:    --font-display  --font-ui  --font-mono
              --radius(-sm/-lg/-pill)  --border-w  --shadow-1  --shadow-2
Иммерсия:     --texture-app  --texture-panel  --texture-btn  --cursor
Движение:     --motion-fast  --motion  --ease
```

Полные значения для Arena и примеры скинов Neon/Tactical — в `tokens.css`.

## 3. Где какой цвет (привязка к функциям приложения)

Дизайн обслуживает реальные сущности комнаты. Цвета закреплены за смыслом:

| Смысл в комнате | Токен | Где видно сейчас |
|-----------------|-------|------------------|
| Главное действие, бренд, фокус | `--accent` | «Показать экран», активная вкладка, кольцо фокуса |
| Голос / микрофон вкл / «на связи» / говорит | `--live` | Кнопка микрофона, рамка+свечение говорящего, статус |
| Хост (корона, права) | `--host` | Значок 👑, «Вернуть права» |
| Кик / бан / ошибка / заглушён хостом | `--danger` | Кнопки «Кикнуть/Забанить», баннеры ошибок, значок 🚫 |
| Предупреждение (нет микрофона, переподключение) | `--warn` | Жёлтые баннеры |
| Инфо / ссылки | `--info` | Подсказки, ссылки |

Правило: **«живое» (зелёное) и «бренд» (акцент) — разные цвета.** Сейчас в коде
всё зелёное (emerald) — и кнопки, и говорящий. Разводим: бренд = ирис,
голос/активность = зелёное. Так интерфейс перестаёт быть монотонным и
подсвечивает именно действие в эфире.

## 4. Типографика

| Роль | Шрифт | Зачем | Кириллица |
|------|-------|-------|-----------|
| Display / заголовки / HUD-подписи | **Exo 2** (600–700) | Техничный, «игровой», но не кричит | да |
| UI / основной текст | **Inter** (400–600) | Эталон читаемости интерфейса | да |
| Моноширинный (код комнаты, тайминги) | **JetBrains Mono** (500–700) | Код комнаты выглядит как «сессия/тег» | да |

Все три есть в Google Fonts с **кириллицей** — критично, интерфейс русский.
Альтернатива для заголовков, если захочется «спортивнее»: **Oswald** (узкий,
как на джерси). Платный скин может подменить `--font-display` на свой
(например, Orbitron для неона, пиксельный — для Minecraft-скина).

**Масштаб (desktop-first), пол — 14px:** H1 28 / H2 18–20 / body 16 / UI 15 /
вторичное 14. Важное (код комнаты, статус, говорящий) — 15px+ и пожирнее. Ниже
14px не опускаемся: правило game-ui-design считает мелкий текст ошибкой
(не читается на дистанции, проваливает доступность). Межстрочный 1.4–1.5.
ALL CAPS — только для коротких HUD-меток (LIVE, «ОТРЯД») с увеличенным
трекингом, не для текста.

## 5. Компоненты (спецификация)

Базовые состояния для каждого: `default / hover / active / focus / disabled`.

- **Кнопки**
  - `primary` — заливка `--accent`, текст `--on-accent`; hover светлее +
    `--glow-accent`; focus — `--ring`; active — `scale(.98)`.
  - `live` — заливка `--live` (микрофон вкл / в эфире).
  - `secondary` — `--surface-2` + рамка `--border`; hover — рамка `--border-strong`.
  - `ghost` — прозрачная (иконки в шапке).
  - `danger` — текст `--danger`; hover — лёгкая заливка `--danger` + рамка.
- **Поле ввода** — фон `--inset`, рамка `--border`, focus — `--ring`. Ник, код,
  пароль, название комнаты.
- **Панель / карточка** — `--surface` + `--border` + `--radius-lg` + `--shadow`.
  Список участников, модалка настроек, карточки «Создать/Войти».
- **Строка участника** — панель `--surface-2`; **говорит** → рамка `--live` +
  `--glow-live` + лёгкая заливка; хост → корона `--host`; заглушён → значок
  `--danger`; справа ползунок громкости (акцентный трек) и кнопка «заглушить».
- **Вкладки (Экран / Доска)** — активная: текст `--accent-hi`, рамка `--accent`,
  лёгкая заливка акцентом.
- **Сцена (демонстрация экрана)** — `--bg-deep`, рамка, скругление; пустое
  состояние «Никто не показывает экран»; в эфире — бейдж `LIVE` (`--danger`).
- **Баннеры** — `info / warn / danger` через одноимённые токены (компонент
  `Banner` уже есть — просто переводим на токены).
- **Модалка** — `--surface-2`, затемнённый backdrop с блюром, шапка с заголовком.
- **Бейджи/значки** — хост (корона), заглушён, «вы», статус соединения.
- **Keycap-подсказки клавиш** — на действиях показываем горячую клавишу
  (`[M]` микрофон, `[D]` заглушить всех, `[S]` показать экран, `[1]/[2]` вкладки).
  Десктоп-аналог «button prompt» из игр; берём актуальную привязку, не хардкод.
- **Лента событий (киллфид)** — очередь уведомлений вверху справа: вход/выход,
  мьют, передача прав. Максимум 2–3 одновременно, авто-затухание, иконка+цвет+
  текст. Критичное (потеря связи) идёт вперёд очереди.
- **Сцена и скос углов** — на области демонстрации «прицельные» уголки-скобки;
  на заливных кнопках и аватарах — скошенный угол (`--clip-hud`). Скин управляет
  агрессивностью: Arena — мягко, Tactical — резко, Neon — без скоса (на свечении).
- **Список участников = табло отряда** — строки-карточки с аватар-плитками,
  именем, статусом и эквалайзером у говорящего. Метафора командного скоринга.
- **Состояния** — у каждого интерактива: `default / hover / active / focus`
  (видимое кольцо) ` / disabled` (тускло + иконка `ti-ban`). Этот же шаблон —
  основа будущих «кулдаунов»: готово / идёт / недоступно.

## 6. Движение

Снаппи и короткое: `--motion-fast` 120мс (hover/нажатия), `--motion` 180мс
(появления/смены), `--ease` `cubic-bezier(.2,.7,.2,1)`. Ключевые анимации:

- **Пульс говорящего** — мягкое дыхание свечения у активной строки.
- **Hover-подъём** кнопок/карточек — едва заметный.
- **Кольцо фокуса** — мгновенное, для доступности.
- Платный иммерсивный скин может усиливать движение (вспышки, плавающий фон).

**Reduced motion (обязательно).** Уважаем системную настройку
`prefers-reduced-motion`: гасим переходы и зацикленные анимации (эквалайзер
замирает, киллфид без слайда), плюс отдельный тумблер «меньше движения» в
настройках. Анимации интерфейса держим <300мс, только ease-out, без слайдов
через пол-экрана и без тряски UI — это требование доступности, не вкусовщина.

## 7. Движок сменяемых тем (архитектура)

### 7.1 Что такое «скин»

Скин = **значения контракта токенов** + (для иммерсивных) **набор ассетов**.
Описывается двумя частями:

1. **CSS** — переопределяет переменные под селектором `[data-skin="id"]`.
2. **Манифест (JSON)** — метаданные и ссылки на ассеты, которые грузит рантайм.

```jsonc
// skins/neon/manifest.json
{
  "id": "neon",
  "name": "Neon Arcade",
  "tier": "paid",                 // free | paid
  "price": 199,
  "css": "/skins/neon/skin.css",  // переопределение токенов
  "fonts": [                      // подгружаются динамически
    { "family": "Orbitron", "google": "Orbitron:wght@500;700" }
  ],
  "assets": {                     // только для иммерсивных скинов
    "textures": { "panel": "/skins/neon/panel.png", "button": "..." },
    "cursor":   { "default": "/skins/neon/cursor.png" },
    "sounds":   {               // звуковой слой UI
      "click": "/skins/neon/click.mp3",
      "join":  "/skins/neon/join.mp3",
      "leave": "/skins/neon/leave.mp3",
      "speak": "/skins/neon/speak.mp3"
    }
  },
  "motion": { "speed": 1.0, "speakingPulse": true }
}
```

### 7.2 Переключение во время работы

```
<html data-skin="arena">   ← рантайм меняет один атрибут
```

CSS-переменные каскадируются → тема меняется мгновенно, без пересборки и
перезагрузки. Бесплатный скин Arena лежит в `:root`, поэтому приложение
работает и без атрибута.

### 7.3 Мост с Tailwind v4

В `src/app/globals.css` маппим утилиты Tailwind на семантические токены —
тогда классы вроде `bg-accent`, `text-live`, `border-default` **сами**
перекрашиваются при смене скина:

```css
@import "tailwindcss";
@import "../../design-system/tokens.css";

@theme inline {
  --color-bg: var(--bg);
  --color-surface: var(--surface);
  --color-surface-2: var(--surface-2);
  --color-accent: var(--accent);
  --color-live: var(--live);
  --color-host: var(--host);
  --color-danger: var(--danger);
  --color-text: var(--text);
  --color-text-dim: var(--text-dim);
  --font-display: var(--font-display);
  --font-sans: var(--font-ui);
  --font-mono: var(--font-mono);
  --radius-DEFAULT: var(--radius);
}
```

> ⚠️ В этом проекте Next.js нестандартный (см. `AGENTS.md`): перед правкой
> framework-файлов сверяться с `node_modules/next/dist/docs/`. Сниппет выше —
> намерение; точный синтаксис `@theme` подтверждаем по докам Tailwind v4 при
> внедрении.

### 7.4 Рантайм `SkinProvider` (звуки, шрифты, курсор)

CSS-переменных хватает для цвета/формы. Иммерсивные слои (шрифты, звуки,
курсор) включает небольшой React-провайдер:

```tsx
// псевдокод намерения
const SkinContext = createContext<{ playSfx: (e: SfxEvent) => void }>(...);

function SkinProvider({ skinId, children }) {
  // 1. ставим data-skin на <html>
  // 2. лениво грузим skin.css и шрифты из манифеста
  // 3. предзагружаем звуки в Audio-пул
  // 4. ставим курсор: document.body.style.cursor = `url(...)`
  // playSfx('click'|'join'|'leave'|'speak') дёргают компоненты на событиях
}
```

У бесплатного Arena звуков/курсора нет — `playSfx` молчит. Громкость звука и
тумблер «звуки интерфейса» кладём в настройки (рядом с громкостью звука,
которая уже хранится в `localStorage`).

### 7.5 Право на скин (монетизация — на будущее)

Бесплатный скин доступен всегда. Платные гейтятся на сервере (вписывается в
уже существующий Upstash Redis — там лежат секреты комнат и rate-limit). Это
последний этап, не блокирует дизайн.

## 8. Дорожная карта внедрения

Порядок такой, чтобы ничего не сломать и двигаться маленькими шагами:

1. **Токены.** Подключить `tokens.css`, добавить мост `@theme inline`
   (раздел 7.3). Внешний вид пока прежний — просто появились переменные.
2. **Перевод компонентов на токены.** Заменить хардкод (`emerald-*`, `zinc-*`)
   на семантические классы/переменные. Начать с `globals.css`, `layout.tsx`
   (шрифты), затем `page.tsx`, `RoomClient.tsx`, `SettingsModal`, `Banner`,
   `TacticsBoard`. После этого Arena уже виден целиком.
3. **Шрифты.** В `layout.tsx` подключить Exo 2 / Inter / JetBrains Mono через
   `next/font` с подмножеством `["latin","cyrillic"]`.
4. **Переключатель скинов + `SkinProvider`.** Сначала просто меняем `data-skin`
   (цвет/форма). UI выбора темы — в настройках.
5. **Второй скин целиком** (например, Neon) — проверка, что движок честно
   работает на «другом мире».
6. **Иммерсивный слой** — пайплайн ассетов: текстуры, звуки, курсор, усиленное
   движение.
7. **Право на скин / магазин** — серверная проверка владения, оплата.

## 9. Что меняет / не меняет скин

| Меняет скин | Не меняет (общий каркас) |
|-------------|--------------------------|
| Цвета (вся семантика) | Структуру экранов и логику |
| Шрифты (display/ui/mono) | Расположение блоков, сетку |
| Радиусы, рамки, тени, свечения | Доступность (фокус, контраст) |
| Текстуры панелей/кнопок/фона | Названия компонентов |
| Курсор, звуки UI, фон-музыку | Поведение LiveKit/комнаты |
| Силу и стиль анимаций | — |

Итог: дизайн — это **сменный «костюм» поверх неизменного каркаса**. Один раз
переведя компоненты на токены, дальше выпускаем темы как контент.

## 10. Игровой HUD и доступность (правила game-ui-design)

Правила взяты из скилла game-ui-design и переведены под веб/десктоп:
**геймпад → клавиатура + фокус**, **safe-zone TV → полноэкранный режим**,
**глифы кнопок → keycap-подсказки**.

- **Читаемость в движении.** Текст поверх видео/текстур — с тенью
  (`--text-shadow-hud`). HUD-элементы закреплены по краям, не прыгают при смене
  сцены. Размер — от 14px, важное крупнее.
- **Цвет никогда не один.** Каждое состояние = цвет + иконка/форма + подпись.
  Говорит = зелёная рамка + свечение + эквалайзер + «говорит…»; хост = золото +
  корона + «хост»; заглушён = алый + `ti-microphone-off` + «микрофон выключен».
  Спасает 8% игроков с дальтонизмом. На будущее — режим «дальтонизм» (сдвиг
  проблемных цветов) и тумблер высокого контраста в настройках.
- **Движение — отключаемо.** `prefers-reduced-motion` + тумблер в настройках;
  переходы <300мс, ease-out, без слайдов через пол-экрана и тряски интерфейса.
- **Клавиатура и фокус (вместо «controller-first»).** Любой элемент доступен с
  клавиатуры; видимое кольцо фокуса на всём; модалка ловит и возвращает фокус;
  горячие клавиши показаны keycap'ами и берутся из текущей привязки, не хардкод.
- **Масштаб интерфейса.** Слайдер `--ui-scale` (75–150%); базовый дизайн 1080p,
  размеры в rem/%, не в фиксированных px — чтобы не «плыло» на 4K и 720p.
- **Слои.** Единая шкала z-index (`--z-dropdown 100 / --z-modal 200 /
  --z-tooltip 300 / --z-notify 400`) — без «войн за 9999».
- **Не захламляем HUD.** Для каждого элемента вопрос: «нужно ли это игроку прямо
  сейчас?». Центр сцены — свято; тулбары и список — по краям.
