# Дизайн: игровые объекты CS2 (гранаты) на тактической доске

> Дата: 2026-06-28. Ветка: `feat/cs2-tactics-board`.
> Это **первая из трёх** подсистем, выделенных при брейншторме улучшений доски.
> Остальные две оформляются отдельными циклами спек → план → реализация:
> **(2) живые курсоры участников**, **(3) снапшоты схем под именем**.

## Цель

Добавить на тактическую доску **game-aware слой** — игровые объекты CS2: гранаты
как иконки/зоны эффекта, которые штампуешь на карту, с опциональной линией
«откуда кидать» и таймингом. Это главный дифференциатор продукта: ни Discord, ни
generic-доска так не умеют.

Первый срез — **гранаты CS2 с линией броска и таймингом**. Каталог объектов
data-driven, чтобы другие игры (RL/Dota) и другие объекты добавлялись данными,
без переделки движка.

## Принцип: новый тип сущности по образцу фигурок/стрелок

Игровой объект — третий тип сущности на доске рядом с `Figure` и `Arrow`.
Переиспользуем всю отлаженную инфраструктуру доски:

- нормированные координаты `0..1` (хелпер `normToRect`, `clamp01`);
- логические часы `epoch` — объекты чистятся вместе со всей доской при «Очистить»;
- снапшот доски при позднем входе/reconnect (`sync-req` → `sync-state`);
- санитайзеры входящих данных (все пакеты приходят от других участников — форме
  доверять нельзя);
- жёсткие лимиты против DoS;
- батч-отправка драга через `requestAnimationFrame`.

Никакого нового канала: тот же data-канал LiveKit, топик `board`.

## Модель данных

Новый модуль `src/lib/boardObjects.ts` (каталог + хелперы) и типы в `src/lib/board.ts`.

```ts
export type ObjKind = "smoke" | "flash" | "molotov" | "he" | "decoy";
export type ObjClass = "zone" | "icon";          // зона эффекта vs точечная иконка
export type Technique = "stand" | "jump" | "runjump" | "move";

export type GameObject = {
  id: string;
  kind: ObjKind;
  x: number; y: number;                 // точка приземления (норм. 0..1)
  radius?: number;                      // только для zone; норм. доля ширины доски
  from?: { x: number; y: number };      // точка броска (опц.) — при наличии рисуется линия
  technique?: Technique;                // чип техники броска (опц.)
  note?: string;                        // свободный текст: «1.6с», «после фейка»
};
```

Каталог типов (цвет/иконка/класс/радиус по умолчанию на тип):

```ts
export type ObjKindDef = {
  kind: ObjKind;
  name: string;          // русское имя для UI
  cls: ObjClass;
  color: string;         // hex
  icon: IconName;        // глиф из game-icons.net
  defaultRadius?: number;// для zone
};

export const CS2_OBJECTS: ObjKindDef[] = [
  { kind: "smoke",   name: "Смок",    cls: "zone", color: "#cdd6df", icon: "nade-smoke",   defaultRadius: 0.07 },
  { kind: "molotov", name: "Молотов", cls: "zone", color: "#ff8a1e", icon: "nade-molotov", defaultRadius: 0.06 },
  { kind: "he",      name: "HE",      cls: "zone", color: "#ffb02e", icon: "nade-he",      defaultRadius: 0.05 },
  { kind: "flash",   name: "Флеш",    cls: "icon", color: "#dcefff", icon: "nade-flash" },
  { kind: "decoy",   name: "Дэкой",   cls: "icon", color: "#34d399", icon: "nade-decoy" },
];
```

Цвет — **по типу гранаты, не по команде**: смок серый, молотов/HE оранжевые, флеш
белый, дэкой бирюзовый. Привязка к CT/T тут не нужна.

Чипы техники (`Technique`): «Стоя» (`stand`), «Прыжок» (`jump`),
«Разбег+прыжок» (`runjump`), «В движении» (`move`).

## Протокол синхронизации

Новые варианты `BoardMessage` (в `src/lib/board.ts`), все несут текущий `epoch`:

- `gobj-add` — **upsert по id**, несёт весь объект: создание, смена типа/техники/
  заметки, коммит финальной геометрии. Идемпотентно (как `fig-add`).
  `{ t: "gobj-add"; epoch: number; obj: GameObject }`
- `gobj-move` — высокочастотный драг, батчем на RAF; патчит присутствующие поля
  геометрии. Покрывает перетаскивание приземления, точки броска и ручки радиуса.
  `{ t: "gobj-move"; epoch: number; id: string; x: number; y: number; fromX?: number; fromY?: number; radius?: number }`
- `gobj-del` — удалить один объект.
  `{ t: "gobj-del"; epoch: number; id: string }`
- `sync-state` — расширяем полем `objects?: GameObject[]`.

Приём:
- `epoch` строго не старше нашей последней очистки (как у фигурок/стрелок);
  больший `epoch` → `catchUpEpoch` (единая очистка всех слоёв + сброс выделения).
- `gobj-add` — upsert по id с потолком `MAX_OBJECTS`.
- `gobj-move` — патч полей объекта с таким id; если объекта нет — игнор (создать
  из move нельзя: нет kind). На reliable-ordered канале `gobj-add` всегда раньше;
  редкий рассинхрон лечит снапшот при reconnect.
- `gobj-del` — фильтр по id.
- В `sync-state` — слияние `objects` по id с потолком `MAX_OBJECTS`, учитываем
  только если снимок не старее нашей последней очистки.

«Очистить» (рост `epoch`) сносит объекты вместе со штрихами/фигурками/стрелками —
`catchUpEpoch` и `clearBoard` дополняются сбросом `objects` и `selectedObjId`.

## Слой рендеринга `GObjectLayer.tsx`

DOM-слой поверх доски, по образцу `FigureLayer` (DOM выбран осознанно: иконки и
круги не искажаются при не-квадратных картах, в отличие от SVG с
`preserveAspectRatio="none"`). Слой меряет свой размер через `ResizeObserver`,
чтобы переводить нормированный `radius` в пиксели.

- **зона** (`cls: "zone"`) — `div` с `radial-gradient` заливкой цвета типа; диаметр
  в px = `radius · ширина_слоя` → круг остаётся кругом при любых пропорциях карты;
  пунктирная обводка. По центру — мелкая иконка типа.
- **иконка** (`cls: "icon"`) — SVG-глиф фиксированного px-размера (не искажается).
- **линия броска** — при наличии `from` рисуем inline-SVG между `from` и `(x,y)`:
  пунктир со стрелкой к точке приземления.
- **подпись тайминга** — у середины линии чип «техника · заметка», если заданы.

Указатель (`pointer-events`) ловится только в режимах «Граната» и «Перемещение»;
иначе слой сквозной (рисуем кистью под объектами). Драг/выделение/удаление по `✕`
повторяют механику `FigureLayer` (`dragId`, флаг `moved`, `stopPropagation`,
нормировка от rect слоя).

**Порядок DOM-слоёв в `TacticsBoard`:** `img` (карта) → `canvas` (штрихи) →
`GObjectLayer` (зоны/иконки/линии) → `ArrowLayer` → `FigureLayer`. Игроки и стрелки
остаются поверх полупрозрачных зон.

## Взаимодействие

- **Поставить объект:** инструмент «Граната» в `BoardRail` → контекстный поповер с
  5 типами (паттерн «контекстный поповер для активного инструмента», как
  цвет/толщина) → выбран тип → клик по доске ставит объект в точку приземления
  (для зон — `defaultRadius`, без `from`).
- **Линия броска:** в режиме «Перемещение» у выделенного объекта есть
  **ручка-хвост**; перетаскивание создаёт `from` и линию. Точка броска затем
  перетаскивается отдельной ручкой.
- **Радиус зоны:** угловая ручка на зоне (диагональное перетаскивание меняет
  `radius`, зажатый в пределах).
- **Тайминг:** двойной клик по объекту/линии → мини-поповер: чипы техники +
  текстовое поле заметки. Изменение шлёт `gobj-add` (upsert).
- **Удалить:** `✕` на выделенном или `Delete`/`Backspace` (если фокус не в поле
  ввода). Существующий обработчик клавиш расширяем третьим типом выделения; выбор
  фигурки/стрелки/объекта остаётся **взаимоисключающим**.

## Безопасность и лимиты (`board.ts`)

Все пакеты приходят от других участников (у всех `canPublishData`) — валидируем
форму перед применением, по образцу `sanitizeFigure`:

- `sanitizeGameObject` / `sanitizeGameObjects`:
  - вайтлист `kind` (`isObjKind`) и `technique` (`isTechnique`);
  - `clamp01` для `x`, `y` и точки `from`;
  - `radius` зажимаем в `0.02..0.25` (для `icon`-типов игнорируем);
  - `note` чистим как `safeLabel`, длина `MAX_NOTE_LEN ≈ 24`;
  - битый/неполный объект → `null` (не роняет отрисовку).
- `MAX_OBJECTS ≈ 40` — потолок и в сообщении, и всего на доске.
- `MAX_ID_LEN` переиспользуем.
- id объектов генерим по тому же принципу, что фигурки/стрелки:
  `${identity}.${mountTag}-obj-${seq}` — без коллизий после ремаунта.

## Что НЕ входит в v1 (YAGNI)

- анимация раскрытия смока / таймлайн-проигрывание схемы;
- дуга-траектория линии броска (только прямая);
- командная привязка гранат (CT/T);

## Как проверить

1. С двух вкладок зайти в комнату. На вкладке A выбрать «Граната» → «Смок» →
   кликнуть по карте: на обеих вкладках появляется зона смока в одной точке.
2. На A перетащить ручку-хвост → появляется линия броска; точка броска и зона
   двигаются — на B всё повторяется в реальном времени.
3. Двойной клик → выбрать чип «Прыжок», вписать «1.6с» → подпись видна на обеих.
4. Поставить флеш (иконка) и HE/молотов (зоны) — рендер по типам, цвета верные.
5. «Очистить» сносит объекты вместе со штрихами/фигурками/стрелками на обеих.
6. Поздний участник (3-я вкладка) после входа получает все объекты снапшотом.
7. Лимиты: попытка превысить `MAX_OBJECTS` не ломает доску; битый пакет игнор.
