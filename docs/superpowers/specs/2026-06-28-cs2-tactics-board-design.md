# Тактическая доска CS2: карты, фигурки, стрелки — дизайн

> Дата: 2026-06-28. Статус: согласовано, готово к плану реализации.
> Контекст: доска тактик ([TacticsBoard.tsx](../../../src/app/room/[code]/TacticsBoard.tsx))
> сейчас умеет только свободную кисть/ластик и фон-картинку по загрузке/ссылке.
> Без встроенных карт и фигурок она «пустая»: чтобы разобрать тактику, надо
> найти и загрузить радар вручную, а игроков рисовать кружочками от руки.
> Цель — убрать это трение и превратить доску в рабочий инструмент для CS2.

## 1. Цель

Три фичи, все — поверх уже работающего протокола доски:

1. **Встроенная библиотека карт CS2** — выбрал из списка, фон встал. Без загрузки.
2. **Фигурки-игроки** — перетаскиваемые токены, цвет по команде (CT/T), подпись
   (номер или ник), синхрон в реальном времени.
3. **Нормальные стрелки** — прямые, сплошная = раш, пунктир = ротация; отдельный
   инструмент, не свободная кисть.

Плюс редизайн тулбара (он перегружается от новых контролов) в боковой HUD-рельс.

Старт — CS2 как игра, в которую играют чаще всего. Архитектура не привязана к
CS2: добавить другую игру = добавить ещё один манифест карт.

## 2. Решения (зафиксированы)

| Решение | Выбор | Почему |
|---|---|---|
| Транспорт фигурок/стрелок | **Существующий data-канал доски** (топик `board`) | Рисунки уже летят там; `epoch`-версионирование, адресный снапшот и строгая валидация переиспользуются. Отдельное хранилище (CRDT) удвоило бы код и разошлось бы с рабочей машинерией. |
| Источник карт | **Публичные community-радары**, вшиты в `public/maps/cs2/` | Пользователь выбрал «найди и вшей сам». Юридически серая зона (ассеты Valve) — см. §9. |
| Смена карты | **Только хост** | Как сейчас загрузка фона. Чтобы карту не перещёлкивали посреди разбора. |
| Модель фигурок | **Свободные токены + палитра** (`+CT`/`+T`) | Не зависит от числа людей в комнате (там 2–6, а состав 5v5). Подпись редактируема. |
| Форма стрелок | **Прямые, 2 точки** | Покрывает 90% случаев; протокол хранит endpoints. Ломаные — возможное расширение позже. |
| Рендер фигурок | **DOM-элементы** поверх canvas | Лёгкий драг, текстовая подпись, доступность, хит-тест. |
| Рендер стрелок | **SVG-слой** | Чёткие наконечники, пунктир, поштучное удаление, хит-тест. |
| Удаление объектов | **Выделить → ✕ / Delete**; стрелки не двигаются в v1 | Минимум UI, прямые дёшево перерисовать. |
| «Очистить» | **Чистит всё** (штрихи+фигурки+стрелки), поднимает `epoch` | Единая семантика версии состояния. |
| Тулбар | **Боковой HUD-рельс** (направление B) + контекстный поповер | Правило дизайн-системы: «центр свят, контролы по краям»; контролы видны только когда нужны. |

## 3. Архитектура: слои доски

Сейчас доска = `<img>` фон + `<canvas>`. Добавляются два слоя сверху. Указатель
маршрутизируется по активному инструменту.

```
  ┌──────────────────────────────────────────────┐
  │ 4. DOM-слой: ФИГУРКИ (div, draggable, label)   │ ← «Перемещение»
  ├──────────────────────────────────────────────┤
  │ 3. SVG-слой: СТРЕЛКИ (vector, marker, dashed)  │ ← «Стрелка» рисует / «Перемещение» выделяет
  ├──────────────────────────────────────────────┤
  │ 2. <canvas>: свободная кисть + ластик          │ ← «Кисть»/«Ластик» (как сейчас)
  ├──────────────────────────────────────────────┤
  │ 1. <img>: фон-карта                            │ (как сейчас)
  └──────────────────────────────────────────────┘
```

Маршрутизация указателя (через `pointer-events` по активному инструменту):

| Инструмент | Кто ловит указатель |
|---|---|
| Перемещение | слой фигурок (драг) + хит-тест по стрелкам (выделение) |
| Кисть / Ластик | canvas |
| Стрелка | svg-слой (рисование резиновой линии) |

Все координаты — нормированные 0..1 относительно рамки доски (как у штрихов),
чтобы совпадать при любых размерах холста и пропорциях карты.

## 4. Протокол: расширение `board.ts`

Новые типы (рядом с `Stroke`):

```ts
export type Team = "ct" | "t";
export type Figure = { id: string; team: Team; label: string; x: number; y: number }; // x,y ∈ 0..1
export type ArrowStyle = "solid" | "dashed";
export type Arrow = { id: string; color: string; style: ArrowStyle; x1: number; y1: number; x2: number; y2: number };
```

Новые сообщения `BoardMessage` (все несут `epoch`, как `stroke`/`clear`):

```ts
| { t: "fig-add";   epoch: number; fig: Figure }
| { t: "fig-move";  epoch: number; id: string; x: number; y: number } // во время драга — батчами через RAF
| { t: "fig-del";   epoch: number; id: string }
| { t: "arrow-add"; epoch: number; arrow: Arrow }
| { t: "arrow-del"; epoch: number; id: string }
```

`sync-state` расширяется полями `figures: Figure[]` и `arrows: Arrow[]` — поздно
зашедший получает полное состояние. Фигурки/стрелки малы (≈40 байт каждая) и
влезают в заголовочный пакет снапшота; батчинг остаётся только для штрихов.

### Валидация и лимиты (в духе существующих `MAX_*`)

Все пакеты приходят от участников (`canPublishData`), поэтому форму не доверяем:

- `MAX_FIGURES = 50`, `MAX_ARROWS = 100`, `MAX_LABEL_LEN = 16`.
- `sanitizeFigure(raw)`: `id` (строка ≤`MAX_ID_LEN`), `team ∈ {ct,t}`,
  `label` — trim + срез до `MAX_LABEL_LEN` + удаление управляющих символов,
  `x,y` через `clamp01`. Иначе `null`.
- `sanitizeArrow(raw)`: `id`, `color` через `safeColor`, `style ∈ {solid,dashed}`,
  4 координаты через `clamp01`. Иначе `null`.
- Применение `fig-add`/`arrow-add` проверяет потолок (`MAX_FIGURES`/`MAX_ARROWS`).
- `epoch` обрабатывается как у штрихов: меньше текущего — игнор; больше — догоняем
  (чистим всё состояние); снапшот мёрджит по `id`.

Тесты — расширить [board.test.ts](../../../src/lib/board.test.ts): новые санитайзеры,
границы лимитов, отбраковка кривого `team`/`style`/координат.

## 5. Библиотека карт

- Радары: `public/maps/cs2/<id>.{jpg|webp}` — Mirage, Dust2, Inferno, Nuke,
  Overpass, Ancient, Anubis, Vertigo, Train. Радары квадратные (1:1) →
  пропорции известны заранее, без скачка вёрстки.
- Манифест `src/lib/maps.ts`:

  ```ts
  export type GameMap = { id: string; name: string; src: string; aspect: number };
  export const CS2_MAPS: GameMap[] = [ { id: "mirage", name: "Mirage", src: "/maps/cs2/mirage.jpg", aspect: 1 }, /* … */ ];
  ```

- **Выбор карты = смена фона** через существующее сообщение `bg`. Единственная
  правка протокола: `sanitizeBgUrl` сейчас пускает только `http(s)`/`data:image`;
  добавить строгий вайтлист корне-относительных путей:
  `^/maps/cs2/[a-z0-9_-]+\.(jpg|png|webp)$` (без `..`, фиксированный префикс —
  тот же origin, безопасно).
- Чип-дропдаун карт — **только хосту** (как контролы загрузки). Не-хост видит имя
  текущей карты статичным чипом. Загрузка своей картинки/ссылки остаётся как есть.

## 6. UX взаимодействия

- **Фигурки:** `+CT`/`+T` на рельсе добавляют токен с авто-номером (1..5) в центр
  доски и шлют `fig-add`. Драг — в режиме «Перемещение» (локально — DOM-transform,
  по сети — `fig-move` батчами через RAF, как точки штриха). Подпись — двойной
  клик по фигурке (дефолт = номер; можно вписать ник).
- **Стрелки:** инструмент «Стрелка» — pointerdown ставит начало, драг рисует
  резиновую прямую (превью локальное), pointerup коммитит и шлёт `arrow-add`
  (одно сообщение, без стриминга — прямая дёшева). Тип сплошная/пунктир — тумблер
  в контекстном поповере (на месте «толщины»); цвет — общий цвет кисти.
- **Удаление:** в «Перемещении» тап по фигурке/стрелке выделяет (подсветка) →
  крестик ✕ + клавиши Delete/Backspace → `fig-del`/`arrow-del`. Стрелки в v1 не
  перетаскиваются.
- **«Очистить»** (рельс) чистит штрихи+фигурки+стрелки и поднимает `epoch`.

## 7. Тулбар: боковой HUD-рельс (направление B)

- Вертикальный рельс, закреплён слева поверх доски (полупрозрачный `--surface` +
  `backdrop-blur`, рамка `--border`): инструменты (Перемещение/Кисть/Ластик/
  Стрелка) · разделитель · `+CT`/`+T` · разделитель · «Очистить».
- Карта — чип в правом верхнем углу доски (хосту — дропдаун, остальным — метка).
- **Контекстный поповер** у рельса, виден только для рисующих инструментов:
  - Кисть → свотчи цвета + толщина;
  - Ластик → толщина;
  - Стрелка → свотчи цвета + тумблер сплошная/пунктир.
- Доступность (правила дизайн-системы, §10 DESIGN_SYSTEM): цели ≥44px, текст
  ≥14px, видимый фокус, `aria-pressed` на инструментах, `prefers-reduced-motion`,
  цвет везде продублирован иконкой/подписью. Команды CT/T — синий/золотой (радар),
  но всегда с буквенной меткой.

## 9. Риски и ограничения

- **Лицензия на карты.** Радары — ассеты Valve; редистрибуция в `public/` — серая
  зона. На старте берём публичные community-радары; при необходимости заменяем на
  собственные схематичные или убираем. Зафиксировано осознанно по выбору владельца.
- **Частота `fig-move`.** Драг может флудить канал — обязателен батчинг через RAF
  (как у точек штриха). На 2–6 участников и ≤50 фигурок нагрузка некритична.
- **Ре-рендер слоёв.** Фигурки/стрелки — в React-state (в отличие от штрихов).
  Локальный драг идёт через DOM-transform/ref, коммит в state — на отпускании;
  удалённые `fig-move` применяются батчами. Для ≤50 объектов это дёшево.

## 10. Порядок реализации (фазы)

1. **Протокол** — расширить `board.ts` (типы, сообщения, санитайзеры, лимиты) +
   тесты. Без UI: фундамент, ничего не ломает.
2. **UI-рельс** — `BoardRail.tsx`, переключение инструментов, контекстный поповер;
   существующая кисть/ластик переезжают в новую раскладку (направление B).
3. **Библиотека карт** — `maps.ts`, ассеты, вайтлист в `sanitizeBgUrl`, чип-дропдаун.
4. **Фигурки** — `useBoardChannel` + `FigureLayer`: add/move/del, драг, подпись, синхрон.
5. **Стрелки** — `ArrowLayer`: рисование, тип сплошная/пунктир, удаление, синхрон.

Каждая фаза самодостаточна и проверяема в комнате (поздний вход, переподключение,
одновременные правки двух участников).
